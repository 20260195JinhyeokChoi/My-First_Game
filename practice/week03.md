# 3주차: 창의프로그래밍입문설계 개발 로그
**프로젝트명:** Survival Blue Ball (가제)
**작성일:** 2026-03-17

## 1. 개요
최초의 기본 이동 로직만 존재하던 파란 공 프로젝트를 바탕으로, 플레이어의 피드백과 기획적 요소를 반영하여 '동적 난이도 조절 시스템'이 도입된 탄막 생존 게임으로 진화시켰습니다.

## 2. 주간 개발 및 변화 과정

### [1단계] 기초 시스템 구축
- **기능:** WASD 및 방향키를 이용한 파란 공의 8방향 이동 구현.
- **연출:** 공 클릭 시 파란색 파티클이 사방으로 튀는 피드백 추가.

### [2단계] 위협 요소 및 피격 판정 도입
- **기능:** 화면 사방에서 플레이어를 추적하여 날아오는 빨간색 탄환 시스템 구현.
- **로직:** 탄환과 공의 거리를 계산하여 충돌 시 공의 크기(`radius`)가 줄어드는 피격 판정 추가.

### [3단계] 게임 루프 및 사망 연출 완성
- **기능:** 공의 크기가 일정 수치 이하로 감소하면 게임 오버 상태로 전환.
- **연출:** 사망 시 파란 파티클이 대량으로 폭발하는 시각 효과 및 재시작(`Restart`) 버튼 구현.
- **UI:** 상단에 현재 FPS와 공의 크기를 실시간으로 표시하는 인터페이스 추가.

### [4단계] 동적 난이도 조절(DDA) 및 시각적 강화
- **기능:** 시간이 경과할수록, 그리고 공의 크기가 작아질수록 탄환 생성 확률이 높아지는 시스템 설계.
- **연출:** 탄환의 이동 경로를 보여주는 빨간색 잔상(Afterimage) 효과 추가 및 피격 파티클 색상 변경을 통한 타격감 개선.

### [5단계] 전략적 요소 및 물리 엔진 확장
- **아이템:** 10%의 확률로 등장하는 초록색 탄환 추가. 획득 시 공의 크기 회복.
- **가변 속도:** 공의 크기가 작아질수록 이동 속도가 빨라지는 물리 법칙 적용 (위기 상황에서의 기동성 확보).

## 3. 최종 결과 요약
단순한 이동 예제에서 시작하여 **위험(빨간 탄환) - 보상(초록 탄환) - 긴장감(시간에 따른 난이도 상승)**이라는 게임의 3요소를 갖춘 생존 게임으로 발전시켰습니다. 특히 플레이어의 상태에 따라 게임 환경이 실시간으로 변하는 시스템을 구현하는 데 집중했습니다.
## 4. 질의응답 및 오류 해결 (Q&A)

### Q1. 프레임 독립적인 이동 구현과 로직 최적화
- **문제:** 컴퓨터 성능에 따라 공의 속도가 달라지거나 탄환 생성이 불규칙해지는 문제 발생 가능성 검토.
- **해결:** `pygame.time.Clock().tick(60)`을 통해 프레임 레이트를 고정하고, `pygame.time.get_ticks()`를 활용하여 프레임이 아닌 '절대 시간' 기준의 난이도 상승 로직을 구현함.

### Q2. 윈도우 환경에서의 CLI 명령어 호환성 오류
- **문제:** 파일 생성 시 유닉스 계열 명령어인 `touch`가 윈도우 커맨드 프롬프트(CMD)에서 인식되지 않는 오류 발생.
- **해결:** `type nul > filename` 또는 `echo` 명령어를 사용하는 윈도우 환경 표준 명령어로 대체하여 개발 환경(OS)에 따른 쉘 명령어의 차이점을 학습함.

### Q3. 투명도(Alpha)가 포함된 레이어 렌더링 이슈
- **문제:** 탄환의 잔상을 구현할 때, 메인 화면(`screen`)에 직접 투명도를 적용하기 어려운 기술적 한계 직면.
- **해결:** `pygame.SRCALPHA` 속성이 적용된 별도의 `Surface` 객체를 생성하여 탄환과 잔상을 먼저 렌더링한 후, 이를 메인 화면에 합성(Blit)하는 레이어 구조 시스템을 도입하여 해결함.

### Q4. 동적 난이도 조절(DDA)의 밸런스 설계
- **문제:** 시간이 지남에 따라 탄환 생성 확률이 무한정 높아져 게임이 불가능해지는 현상 우려.
- **해결:** `min()` 함수를 사용하여 시간 가중치에 상한선(Cap)을 설정하고, 공의 크기가 커질 경우 난이도가 다시 완화되는 유동적 확률 계산식을 적용하여 게임 플레이의 긴장감을 유지함.
