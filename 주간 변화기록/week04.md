# 4주차: 창의프로그래밍입문설계 개발 로그
**프로젝트명:** Survival Blue Ball (가제) - 충돌 판정 시각화 테스트
**작성일:** 2026-03-24

## 1. 개요
기존의 고정된 충돌 판정에서 벗어나, 오브젝트가 회전할 때 발생하는 판정 오차를 이해하고 이를 해결하기 위한 다양한 충돌 모델(Circle, AABB, OBB)을 동일한 환경에서 구현하여 비교 분석하였습니다.

## 2. 주간 개발 및 변화 과정

### [1단계] 이미지 회전 및 중심점 고정
- **내용:** 칼(Sword) 에셋을 불러와 매 프레임 일정 각도로 회전하도록 구현.
- **해결:** 이미지 회전 시 크기 변화로 인해 발생하는 위치 어긋남 문제를 `get_rect(center=...)` 메서드를 사용하여 중심점을 고정함으로써 해결함.

### [2단계] 3가지 충돌 범위 시각화
- **Circle (파란색):** 중심점 기준의 반지름 판정. 계산은 빠르나 긴 형태의 물체에는 부적합함을 확인.
- **AABB (빨간색):** 축에 고정된 사각형 판정. 물체가 회전함에 따라 판정 박스가 불필요하게 커지는 '여백 문제'를 관찰함.
- **OBB (초록색):** 물체의 회전 각도를 그대로 반영하는 사각형 판정. 실제 이미지의 외형과 가장 흡사한 판정 영역을 생성함.

### [3단계] 실시간 판정 피드백 시스템
- **내용:** 각 충돌 방식별로 대상(회색 상자)과 겹쳤을 때 "HIT" 메시지가 출력되도록 구현.
- **성과:** 회전하는 칼날이 시각적으로는 닿지 않았음에도 AABB 판정에서는 '충돌'로 인식되는 오차 범위를 직접 확인하고, OBB의 정밀도를 검증함.

## 3. 최종 결과 요약
동일한 회전 오브젝트에 대해 세 가지 충돌 알고리즘을 동시에 적용해 봄으로써, 게임 기획 의도에 맞는 적절한 충돌 모델 선택의 중요성을 학습했습니다. 특히 OBB 구현을 통해 회전이 잦은 근접 무기 등에 필요한 정밀 판정의 기초 원리를 파악했습니다.

## 4. 주요 해결 과제 (Q&A)

### Q1. 이미지와 히트박스의 회전 동기화 문제
- **현상:** 칼 이미지는 시계 방향으로 도는데, 직접 계산한 OBB 좌표는 반대로 도는 현상 발생.
- **해결:** 파이게임 좌표계의 특성을 고려하여 삼각함수 계산 시 각도 부호를 조정, 시각적 이미지와 물리적 판정 영역을 일치시킴.

### Q2. 상황별 판정 방식의 효율성 비교
- **분석:** 무조건 정밀한 OBB가 좋은 것이 아니라, 연산 비용을 고려하여 탄환처럼 작은 물체는 Circle을, 회전이 없는 건물 등은 AABB를, 정밀한 검술 액션 등에는 OBB를 사용하는 것이 효율적임을 이해함.
